We are implementing CSV export and plotting for the balls-into-bins simulation project.

Do NOT change the core simulation algorithms.
Do NOT change metric computation unless absolutely required.
Keep using namespace balls_bins in C++.
Do not add `using namespace std;`.

Current project state:
- SimulationBase computes per-trial metrics and aggregated metrics.
- SimulationMetrics contains max_load, cv_load, cost_per_ball, and cost breakdown fields.
- SimulationReferences contains reference helpers:
  - unweighted optimal max load
  - unweighted optimal CV
  - weighted average-load lower bound
- Allocation RNG and weighted workload RNG are deterministic.

Goal:
Create:
1. A C++ experiment runner that exports one CSV.
2. A Python plotting script that reads the CSV and generates the final plots.

Use C++ for simulation + CSV export.
Use Python + matplotlib for plotting.
Do NOT use seaborn.

- results/
- plots/

Reference behavior:
- For unweighted:
  reference_type = "unweighted_optimal"
  reference_max_load = ceil(m/n)
  reference_cv_load = optimal CV from floor/ceil allocation
  reference_note = "Optimal unweighted allocation"

- For weighted:
  reference_type = "weighted_average_lower_bound"
  reference_max_load = average_load_mean
  reference_cv_load = 0.0
  reference_note = "Average-load lower bound only; true weighted optimum requires partition/bin-packing"

Global constants:
- balls = 10000
- bins values: 16 and 32
- trials = 30 by default. If runtime is too slow, use 10.
- weighted max weight = 10.0
- workload_seed = 42
- allocation_seed = 1337
