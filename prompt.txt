1. Add deterministic allocation RNG seed

Currently workload_rng_ is seeded deterministically, but allocation RNG uses std::random_device.
Add a new constructor parameter to SimulationBase:

    unsigned int allocation_seed = 1337

Store it as:

    unsigned int allocation_seed_;

Expose:

    unsigned int getAllocationSeed() const;

Update all derived simulator constructors:
- PowerKSimulator
- StatefulRoundRobinSimulator
- HeapSizeSPowerOfKSimulator

They should accept allocation_seed with the same default value and pass it to SimulationBase.

Important:
- Do not break existing call sites. Keep defaults so current code compiles.
- In SimulationBase::run(), for each trial, seed allocation RNG deterministically:

    rng_.seed(allocation_seed_ + static_cast<unsigned int>(trial));

- Keep workload_rng_ seeding as it is:

    workload_rng_.seed(workload_seed_ + static_cast<unsigned int>(trial));

This ensures:
- weighted ball sequence is deterministic
- random bin choices are deterministic
- repeated runs produce identical CSV/graph data

2. Add reference baseline helpers

Create a small new header/source pair if appropriate:

    SimulationReferences.h
    SimulationReferences.cpp

or add this to SimulationMetrics if you prefer, but keep the code clean.

Add:

    enum class ReferenceType {
        None,
        UnweightedOptimal,
        WeightedAverageLowerBound
    };

    struct ReferenceMetrics {
        ReferenceType type = ReferenceType::None;
        double max_load = 0.0;
        double cv_load = 0.0;
    };

Add helper functions:

    double computeUnweightedOptimalMaxLoad(int balls, int bins);

Should return:

    ceil(static_cast<double>(balls) / bins)

and handle balls == 0 correctly.

Add:

    double computeUnweightedOptimalCvLoad(int balls, int bins);

Definition:
- For unweighted balls, the optimal allocation places:
  - remainder bins with ceil(m/n)
  - the rest with floor(m/n)
- Compute the population standard deviation of this optimal load vector.
- Divide by the average load.
- If average load is zero, return 0.

This is important because if m is not divisible by n, optimal CV is not zero.

Add:

    ReferenceMetrics computeReferenceMetrics(int balls,
                                             int bins,
                                             bool weighted_balls,
                                             double average_load);

Behavior:
- If weighted_balls == false:
    type = ReferenceType::UnweightedOptimal
    max_load = computeUnweightedOptimalMaxLoad(balls, bins)
    cv_load = computeUnweightedOptimalCvLoad(balls, bins)

- If weighted_balls == true:
    type = ReferenceType::WeightedAverageLowerBound
    max_load = average_load
    cv_load = 0.0 or NaN, whichever is simpler and safer for CSV export later.
    Prefer 0.0 if you do not want to handle NaN in CSV.

Comment clearly:
- Weighted average load is only a lower bound on max_load.
- The true weighted optimum requires solving a partition/bin-packing style problem and is not computed here.
