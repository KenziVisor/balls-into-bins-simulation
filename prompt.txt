Goal:
Implement a new simulator class:

HeapSizeSPowerOfKSimulator

This class models a partial persistent heap policy.

Definitions:

* n = number of bins
* m = number of balls
* s = persistent heap size, i.e. number of bins tracked in the heap
* k = number of bins sampled from the untracked bins on each placement decision

Policy intuition:
The simulator maintains a persistent heap of s tracked bins.
For each ball:

1. Optionally inspect k bins from outside the heap.
2. Find the minimum-load inspected untracked bin.
3. Try to admit that candidate into the heap.
4. Place the ball into the minimum-load bin currently tracked by the heap.
5. Update the heap entry after placement.

Validation:

* 1 <= s <= n
* If s < n:

  * 1 <= k <= n - s
* If s == n:

  * k is irrelevant because all bins are tracked.
  * Allow k == 0.
  * Do not require k >= 1 in this case.
  * Do not sample outside the heap.
  * Do not charge random_draw, outside load reads, or outside comparisons.
* Throw std::invalid_argument for invalid configurations.

Important heap ordering:
C++ heap helpers create max-heaps by default, so be careful.
We need the policy to retrieve the minimum-load tracked bin.
Use a comparator or helper functions so that:

* Smaller load is better.
* If loads are equal, smaller bin_index is better.
* The heap root should represent the minimum tracked bin.

Keep deterministic tie-breaking.

Trial initialization:
At the beginning of every runSingleTrial():

* Clear heap_.
* Reset is_tracked_ to size n_, all false.
* Initialize the heap with bin indices 0, 1, ..., s_ - 1.
* Mark these bins as tracked.
* Initial loads are 0.0 because SimulationBase::reset() has already cleared bins_.
* Charge persistent state memory once per trial:
  total_cost_ += static_cast<double>(s_) * cost_weights_["state_memory_per_slot"];

Cost model:
The total_cost_ metric measures only operational placement-policy cost.
Do not charge any cost for drawing ball weights.
Do not add a "weight_draw" cost.
Do not reuse "random_draw" for ball weights.

Use existing cost keys:

* "random_draw": randomly sampling an untracked candidate bin
* "load_read": reading actual bin load from bins_
* "compare": load comparisons
* "state_read": reading persistent heap/tracked state
* "state_update": modifying persistent heap/tracked state
* "state_memory_per_slot": persistent heap memory

Add a new default cost key to SimulationBase:

* "heap_update_per_level" with default value 1.0

Add a private helper:
int heapUpdateLevels() const;

Return:

* 1 if s_ <= 1
* otherwise ceil(log2(s_ + 1))

Whenever the heap is rebuilt/reordered after an update, charge:
total_cost_ += static_cast<double>(heapUpdateLevels()) * cost_weights_["heap_update_per_level"];

Do not over-engineer exact heap internals. The goal is a logical cost model that captures O(log s) heap update behavior.

Untracked sampling logic:
If s_ == n_:

* Skip all outside sampling/admission logic.
* Do not charge random_draw.
* Do not charge outside load_read.
* Do not charge outside compare.
* The policy is a full persistent heap over all bins.

If s_ < n_:
There are n_ - s_ untracked bins.

Special case: k_ == n_ - s_

* This means we inspect all untracked bins.
* Do not charge random_draw, because this is not random sampling.
* Iterate over all currently untracked bins.
* For each untracked bin:

  * read its load using readBinLoad(bin), so load_read is charged.
* Use comparison charging to find the minimum-load untracked bin.
* This models checking all untracked bins, not randomly sampling them.

General case: k_ < n_ - s_

* Randomly sample k_ distinct bins from the currently untracked bins.
* Do not use drawRandomBins(k_) directly, because it samples from all bins and may include tracked bins.
* Implement a helper that samples from untracked bins only.
* Charge:
  total_cost_ += static_cast<double>(k_) * cost_weights_["random_draw"];
* For each sampled bin:

  * read its current load using readBinLoad(bin)
* Use comparison charging to find the minimum sampled untracked bin.

Admission logic:
After finding the best untracked candidate:

* Find the heaviest currently tracked bin in heap_.
* For simplicity, scan heap_.
* When scanning heap_:

  * charge state_read for each heap entry inspected
  * charge compare for comparisons
* The heaviest tracked bin is the worst tracked entry:

  * larger load is worse
  * if loads are equal, evict the larger bin_index for deterministic behavior
* Admit the candidate if:
  candidate_load <= heaviest_tracked_load
* Use <= intentionally so that early in the simulation, when many loads are equal, untracked bins can enter the heap.
* If admitted:

  * Mark the evicted bin as untracked in is_tracked_
  * Mark the candidate bin as tracked
  * Replace the heaviest heap entry with the candidate
  * Charge state_update for modifying persistent state
  * Rebuild/reorder heap_
  * Charge heap_update_per_level * heapUpdateLevels()
* If not admitted:

  * Do not modify heap_
  * Do not charge state_update

Placement logic per ball:
For each ball:

1. Draw the ball weight using drawBallWeight().

   * This must not charge cost.
2. If s_ < n_, perform outside sampling and admission.

   * If s_ == n_, skip this entirely.
3. Read the minimum tracked bin from the heap root.

   * Charge state_read.
4. Add the ball to that bin using addBallToBin(min_bin, ball_weight).
5. Update the heap root load by adding ball_weight.

   * Charge state_update.
6. Reorder heap_ after the root load changed.

   * Charge heap_update_per_level * heapUpdateLevels().

Important:
The canonical bin loads are still bins_ in SimulationBase.
The heap stores a policy-state copy of the tracked loads.
Whenever a ball is added to a tracked bin, update both:

* bins_ using addBallToBin()
* the corresponding HeapEntry load
