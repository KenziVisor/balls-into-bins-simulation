We need to fix the weighted-balls workload generation in the existing C++ balls-into-bins simulation.

Current structure:

* SimulationBase is the abstract base class.
* PowerKSimulator inherits from SimulationBase.
* StatefulRoundRobinSimulator inherits from SimulationBase.
* The project uses namespace balls_bins.
* Do not use `using namespace std;`.
* Keep the code simple and interview-readable.

Problem:
Weighted balls currently introduce experimental noise if every simulator generates its own unrelated sequence of weights.

Goal:
All simulator policies should run on the same ball-weight sequence when they use the same:

* m
* weighted_balls flag
* max_weight
* workload_seed
* trial index

This should be done without storing all ball weights in a vector as part of the simulator state.

Design:
Separate the randomness used for workload generation from the randomness used by placement policies.

In SimulationBase:

* Keep the existing RNG for policy randomness, or rename it to policy_rng_ if convenient.
* Add a separate RNG for workload generation:
  std::mt19937 workload_rng_;
* Add a workload seed:
  unsigned int workload_seed_;

Update the SimulationBase constructor signature to include a workload seed:

SimulationBase(
int m,
int n,
int trials = 1,
bool weighted_balls = false,
double max_weight = 1.0,
unsigned int workload_seed = 42
);

If the existing constructor already includes weighted_balls and max_weight, only add workload_seed after them.

Validation:

* max_weight_ must be >= 1.0.
* workload_seed_ should be stored as provided.

Add a public getter:
unsigned int getWorkloadSeed() const;

Important behavior:
At the beginning of every trial, before runSingleTrial() is called, reset the workload RNG to a deterministic trial-specific seed:

workload_rng_.seed(workload_seed_ + static_cast<unsigned int>(trial));

This ensures that:

* Trial 0 always gets the same workload sequence for the same workload_seed.
* Trial 1 gets a different but deterministic workload sequence.
* Different policies get identical ball weights for the same trial when configured with the same workload_seed.

drawBallWeight():

* Must use workload_rng_, not the policy RNG.
* If weighted_balls_ is false, return 1.0.
* If weighted_balls_ is true, draw from std::uniform_real_distribution<double>(1.0, max_weight_).
* drawBallWeight() must never modify total_cost_.
* Do not add a weight_draw cost key.
* Do not reuse random_draw cost for ball weights.

Policy randomness:

* drawRandomBins() must continue to use the policy RNG, not workload_rng_.
* random_draw cost should still apply only to random bin selection.

Update derived constructors:
PowerKSimulator constructor should accept and forward workload_seed:

PowerKSimulator(
int m,
int n,
int k,
int trials = 1,
bool weighted_balls = false,
double max_weight = 1.0,
unsigned int workload_seed = 42
);

StatefulRoundRobinSimulator constructor should accept and forward workload_seed:

StatefulRoundRobinSimulator(
int m,
int n,
int trials = 1,
bool weighted_balls = false,
double max_weight = 1.0,
unsigned int workload_seed = 42
);

Verification support:
Add a public method to SimulationBase:

std::vector<double> previewBallWeights(int count, int trial_index = 0) const;

Behavior:

* It should not modify the simulator’s actual workload_rng_ state.
* It should create a local std::mt19937 preview_rng seeded with:
  workload_seed_ + static_cast<unsigned int>(trial_index)
* If count is negative, throw std::invalid_argument.
* If weighted_balls_ is false, return count values of 1.0.
* If weighted_balls_ is true, use preview_rng to generate count weights from [1.0, max_weight_].
* It must not modify total_cost_.
* It should be used only for reproducibility/debug verification.

main.cpp verification:
Add an explicit sanity check in main.cpp before running the simulation summaries.

Implement a helper function in main.cpp, for example:

Important:

* This check should compare a weighted PowerKSimulator and a weighted StatefulRoundRobinSimulator configured with the same max_weight and workload_seed.
* It should print "Workload preview match: yes" when the implementation is correct.
* This verification is intentionally lightweight and may remain in main.cpp for now.
* Do not store all 10000 weights in the simulator.
* Do not change the meaning of total_cost_.
* Do not charge cost for previewing or drawing weights.
* Do not change the unweighted behavior.
* Keep all code inside namespace balls_bins.
* Update headers, cpp files, and Makefile if needed.
