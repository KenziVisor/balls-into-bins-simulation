We are preparing the balls-into-bins simulator for graph generation later, but do NOT implement plotting yet.

Current code structure:
- SimulationBase is the abstract base class.
- PowerKSimulator implements random / power-of-k / absolute minimum.
- StatefulRoundRobinSimulator implements round-robin.
- HeapSizeSPowerOfKSimulator implements a stateful heap-based policy.
- main.cpp currently prints summaries directly.
- Keep using the balls_bins namespace. Do not add `using namespace std;`.

Goal:
Refactor metric collection so every simulation produces clean per-trial metrics and aggregated metrics. This is needed before we later edit main.cpp to generate CSV files and graphs.

Important design requirement:
Do NOT compute max/min/stddev over averaged bin loads. Instead, compute all metrics per trial, then aggregate each metric across trials.

Required changes:

1. Add a new metrics type, preferably in a new file:
   - SimulationMetrics.h
   - If implementation helpers are needed, add SimulationMetrics.cpp.

Create a struct TrialMetrics with at least:

- double average_load;
- double min_load;
- double max_load;
- double max_min_gap;
- double stddev_load;
- double cv_load;
- double normalized_max_load;
- double normalized_gap;
- double p90_load;
- double p95_load;
- double total_cost;
- double cost_per_ball;

Also include enough cost breakdown fields to support future cost-vs-quality graphs:

- double random_draw_cost;
- double load_read_cost;
- double compare_cost;
- double state_read_cost;
- double state_update_cost;
- double state_memory_cost;
- double heap_update_cost;

And optionally:

- double stateless_cost;
- double stateful_cost;

where:
- stateless_cost = random_draw_cost + load_read_cost + compare_cost
- stateful_cost = state_read_cost + state_update_cost + state_memory_cost + heap_update_cost

2. Add an AggregatedMetrics struct that stores mean and sample standard deviation across trials for the important metrics.

At minimum aggregate:
- average_load
- min_load
- max_load
- max_min_gap
- stddev_load
- cv_load
- normalized_max_load
- normalized_gap
- p90_load
- p95_load
- total_cost
- cost_per_ball
- every cost breakdown field listed above

Prefer a compact reusable representation if it keeps the code simple, for example:

struct MetricSummary {
    double mean;
    double stddev;
};

struct AggregatedMetrics {
    MetricSummary max_load;
    MetricSummary normalized_max_load;
    ...
};

3. Add helper functions:
- TrialMetrics computeTrialMetrics(const std::vector<double>& bins,
                                   double total_cost,
                                   int balls,
                                   const CostBreakdown& cost_breakdown);

- AggregatedMetrics aggregateMetrics(const std::vector<TrialMetrics>& trials);

Make percentile calculation deterministic and simple:
- sort a copy of the bin loads
- use nearest-rank or ceil-based percentile
- document the chosen method in a comment

4. Refactor SimulationBase to store metrics.

Currently SimulationBase stores:
- bins_
- total_cost_

Keep these, but add:
- std::vector<TrialMetrics> trial_metrics_;
- AggregatedMetrics aggregated_metrics_;
- CostBreakdown cost_breakdown_;

Expose getters:
- const std::vector<TrialMetrics>& getTrialMetrics() const;
- const AggregatedMetrics& getAggregatedMetrics() const;
- const CostBreakdown& getCostBreakdown() const;

Important:
- reset() should reset bins_, total_cost_, and cost_breakdown_ for the current trial.
- run() should:
  a. clear trial_metrics_
  b. for each trial:
     - reset()
     - seed workload_rng_ as before
     - runSingleTrial()
     - compute TrialMetrics from that trial's final bins and costs
     - push it into trial_metrics_
     - accumulate bins only for optional averaged final loads / debugging
  c. compute aggregated_metrics_ from trial_metrics_
  d. keep bins_ as averaged final loads only for backward compatibility/debugging
  e. keep total_cost_ as mean total cost across trials for backward compatibility

5. Refactor cost accounting.

Right now total_cost_ is incremented directly in multiple places. Keep total_cost_, but also track cost by category.

Create:

struct CostBreakdown {
    double random_draw = 0.0;
    double load_read = 0.0;
    double compare = 0.0;
    double state_read = 0.0;
    double state_update = 0.0;
    double state_memory = 0.0;
    double heap_update = 0.0;

    double total() const;
    double statelessCost() const;
    double statefulCost() const;
};

Add protected helper methods in SimulationBase:
- void addRandomDrawCost(double units);
- void addLoadReadCost(double units);
- void addCompareCost(double units);
- void addStateReadCost(double units);
- void addStateUpdateCost(double units);
- void addStateMemoryCost(double units);
- void addHeapUpdateCost(double units);

Each helper should multiply units by the matching cost weight, update cost_breakdown_, and update total_cost_.

Then replace direct `total_cost_ += ...` usage in all simulators with the helper methods.

Examples:
- drawRandomBins(k): addRandomDrawCost(k)
- readBinLoad(): addLoadReadCost(1)
- lessThan(): addCompareCost(1)
- StatefulRoundRobinSimulator state read/update/memory should use addStateReadCost/addStateUpdateCost/addStateMemoryCost
- HeapSizeSPowerOfKSimulator heap update should use addHeapUpdateCost(levels)
- comparisons inside HeapSizeSPowerOfKSimulator should use addCompareCost(1)

6. Preserve existing behavior.

Do not change the allocation algorithms.
Do not change tie-breaking behavior.
Do not remove weighted ball support.
Do not implement CSV export yet.
Do not implement plotting yet.
Do not redesign main.cpp yet, except minimally to compile with the new metrics API.

7. Update printSimulationSummary in main.cpp to use AggregatedMetrics.

For now, print the same general summary, but use aggregated metrics instead of recomputing metrics from sim.getBins().

Print at least:
- Average load mean
- Max load mean ± stddev
- Normalized max load mean ± stddev
- Max-min gap mean ± stddev
- CV mean ± stddev
- P90/P95 load mean
- Cost per ball mean ± stddev
- Cost breakdown per ball:
  - random draw
  - load read
  - compare
  - state read/update/memory
  - heap update

Keep printing averaged final bin loads optional or remove it from the main output to reduce noise. If you keep it, label it as debug/backward-compatible output.

8. Validation expectations.

After the refactor:
- `make` should compile cleanly with -std=c++17 -Wall -Wextra -pedantic.
- Running the program should still execute all existing simulation scenarios.
- For trials = 1, stddev across trials should be 0.
- For unweighted balls, average_load should be exactly or very close to m/n.
- normalized_max_load should be max_load / average_load.
- normalized_gap should be (max_load - min_load) / average_load.
- cost_per_ball should be total_cost / m, with 0 when m == 0.

9. Keep code style simple.

Prefer readable C++17 over clever abstractions.
Use std::vector, std::string, std::unordered_map only where needed.
Avoid global mutable state.
Keep all new types inside namespace balls_bins.
