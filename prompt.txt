5. Experiment definitions

Create scenarios for the following experiments.

Experiment 1: Stateless baseline
experiment_id = "01_stateless_baseline"

For n=32, k values:
- 1, 2, 3, 4, 8, 16, 32

family = "stateless"

Experiment 3: Stateful vs stateless under equal weights
experiment_id = "03_stateful_vs_stateless_unweighted"
Workload: unweighted
Policies for both n=16 and n=32:
- Random: PowerKSimulator k=1
- Power-2: PowerKSimulator k=2
- Power-3: PowerKSimulator k=3
- Round-robin: StatefulRoundRobinSimulator
- Absolute minimum: PowerKSimulator k=n

Experiment 4: Weighted balls break round-robin
experiment_id = "04_weighted_round_robin_break"
Workload: weighted, max_weight=10
Policies for both n=16 and n=32:
- Random weighted: PowerKSimulator k=1
- Power-2 weighted: PowerKSimulator k=2
- Power-3 weighted: PowerKSimulator k=3
- Round-robin weighted: StatefulRoundRobinSimulator
- Absolute minimum weighted: PowerKSimulator k=n
Do NOT include heap in this experiment.

Experiment 5: Heap size sweep
experiment_id = "05_heap_sweep"
Workload: weighted, max_weight=10
Use HeapSizeSPowerOfKSimulator only.
Important:
- For partial heap, fix k=2.
- Do not sweep k.
- k=2 is the conclusion from the earlier power-of-k experiments.
For n=16, heap sizes:
- s=2,4,8,16
For n=32, heap sizes:
- s=2,4,8,16,32
For s<n:
- use k=2
For s==n:
- full heap, use k=0

Experiment 6: Final tradeoff unweighted
experiment_id = "06_final_tradeoff_unweighted"
Workload: unweighted
Policies for both n=16 and n=32:
- Random: PowerKSimulator k=1, family stateless
- Power-2: PowerKSimulator k=2, family stateless
- Power-3: PowerKSimulator k=3, family stateless
- Power-4: PowerKSimulator k=4, family stateless
- Absolute minimum: PowerKSimulator k=n, family stateless
- Round-robin: StatefulRoundRobinSimulator, family round_robin
- Partial heap s=4 k=2: HeapSizeSPowerOfKSimulator, family heap
- Partial heap s=8 k=2: HeapSizeSPowerOfKSimulator, family heap
- Partial heap s=16 k=2: HeapSizeSPowerOfKSimulator, family heap
- Full heap: HeapSizeSPowerOfKSimulator s=n k=0, family heap

Important:
- If heap size s is larger than n, skip that scenario.
  Example: for n=16, s=16 is valid and is also full-size state. Avoid duplicating identical labels if needed.

Experiment 7: Final tradeoff weighted
experiment_id = "07_final_tradeoff_weighted"
Workload: weighted, max_weight=10
Same policy list as Experiment 6.
For partial heap, always use k=2.
For full heap, use s=n, k=0.

7. Python plotting script

Create:

    plot_results.py

It should read:

    results/simulation_results.csv

and generate PNG files under:

    plots/

Use:
- pandas
- matplotlib
- no seaborn

If pandas is not available, it is acceptable to implement simple CSV loading, but pandas is preferred.

General plotting rules:
- Every line must be separated by both color and marker.
- Use clear titles, axis labels, legends, and readable tick labels.
- Save at high resolution, e.g. dpi=200.
- Use tight_layout.
- Do not use tiny fonts.
- Rotate categorical x-axis labels when needed.
- Include error bars when stddev is meaningful and does not clutter the chart.
- Prefer one figure per PNG.

For experiments 1-5:
Each metric graph should have two vertical panels sharing the same x-axis:
- Top panel: quality metric, either max_load or cv_load
- Bottom panel: cost_per_ball

For experiments 6-7:
Each graph is a scatter plot:
- X-axis: cost_per_ball_mean
- Y-axis: max_load_mean or cv_load_mean
- color: policy_family
- marker: bins, one marker for n=16 and another for n=32
- annotate points with short policy labels if readable; otherwise use legend only.
