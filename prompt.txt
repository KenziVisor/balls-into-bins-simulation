Review and fix the current HeapSizeSPowerOfKSimulator implementation.

Current issue:
The simulator currently reconstructs the list of untracked bins by scanning all n bins inside sampleUntrackedBins() and in the full-outside-scan path.

This is not acceptable for the policy logic. The simulator should logically know which bins are inside the heap and which bins are outside the heap as part of the heap policy state.

Important modeling decision:
Knowing heap membership is part of the heap abstraction.
Do not charge additional logical memory cost for maintaining membership/indexing structures.
The logical persistent memory cost remains:

s_ * cost_weights_["state_memory_per_slot"]

Do not add extra memory costs for helper vectors or position maps.

Goal:
Avoid logically O(n) work when k < n - s.
Sampling k untracked bins should be O(k), not O(n).

Required design:
In HeapSizeSPowerOfKSimulator, add and maintain:

std::vector<int> untracked_bins_;
std::vector<int> heap_positions_;
std::vector<int> untracked_positions_;

Semantics:

* untracked_bins_ stores all bins currently outside the heap.
* heap_positions_[bin] is the current index of bin inside heap_, or -1 if bin is not tracked.
* untracked_positions_[bin] is the current index of bin inside untracked_bins_, or -1 if bin is tracked.

These structures are part of the heap/membership representation and should not add logical memory cost.

Trial initialization:
In initializeTrialState():

* Clear heap_ and untracked_bins_.
* Resize heap_positions_ and untracked_positions_ to n_ and initialize all entries to -1.
* Add bins 0..s_-1 to heap_.
* Set heap_positions_[bin] for those bins.
* Add bins s_..n_-1 to untracked_bins_.
* Set untracked_positions_[bin] for those bins.
* Build the heap.
* Charge only:
  total_cost_ += static_cast<double>(s_) * cost_weights_["state_memory_per_slot"];

Do not use std::vector<bool> is_tracked_ anymore unless you keep it only as a redundant debug aid. Prefer removing it to avoid inconsistent state.

Sampling logic:
Update sampleUntrackedBins(int count):

* It must not scan all n bins.
* It should sample count distinct bins from untracked_bins_ using partial shuffle over untracked_bins_ indices.
* It should not permanently reorder untracked_bins_ unless that is intentional and positions are updated correctly.
* Simpler preferred implementation:

  * Create a local vector<int> sampled_indices containing 0..untracked_bins_.size()-1
  * Partial shuffle sampled_indices for count entries
  * Return untracked_bins_[sampled_indices[i]]
    This still allocates O(n-s), so avoid that too if possible.

Better implementation:

* Use a local std::unordered_set<int> or vector of sampled positions to sample distinct positions from [0, untracked_bins_.size()-1] until count positions are selected.
* Since count == k and this path only applies when k < n-s, this is logically O(k) expected time.
* Charge:
  total_cost_ += static_cast<double>(count) * cost_weights_["random_draw"];
* Return the sampled bin indices.
* Do not scan all n bins.

Full outside scan:
If k_ == n_ - s_:

* Iterating over all untracked_bins_ is correct and intentional.
* Do not charge random_draw.
* For each bin in untracked_bins_:

  * read its load using readBinLoad(bin)
  * compare to find the minimum
    This models checking all untracked bins.

Admission update:
When a candidate is admitted:

* Let evicted_bin be the bin currently in the heaviest heap entry.
* Replace evicted_bin with candidate_bin in heap_.
* Update heap_positions_:

  * heap_positions_[evicted_bin] = -1
  * heap_positions_[candidate_bin] = heap index where it was inserted
* Update untracked_bins_ and untracked_positions_ so that:

  * candidate_bin is removed from untracked_bins_
  * evicted_bin is added to untracked_bins_
    This update must be O(1).

Implement helper methods if useful:

* void removeFromUntracked(int bin)
* void addToUntracked(int bin)
* void updateHeapPositions()
* void rebuildHeapAndPositions()

For O(1) removeFromUntracked:

* Find the index from untracked_positions_[bin].
* Swap that element with the last element of untracked_bins_.
* Update untracked_positions_ for the swapped bin.
* pop_back().
* Set untracked_positions_[bin] = -1.

For addToUntracked:

* Set untracked_positions_[bin] = untracked_bins_.size().
* push_back(bin).

Heap positions:
Because reorderHeap() calls std::make_heap(), heap entry positions may change.
After every reorderHeap(), update heap_positions_ by scanning heap_ once.
This scan is an implementation detail of maintaining heap positions after reorder.
Do not charge extra logical cost for updating helper position metadata.
The logical heap update cost is already charged by:
heapUpdateLevels() * cost_weights_["heap_update_per_level"]

Cost model:

* Do not charge extra costs for maintaining heap_positions_ or untracked_positions_.
* Do not charge extra memory for untracked_bins_ or positions.
* Keep charging:

  * random_draw only for random sampling of untracked candidates
  * load_read for reading actual bin loads
  * compare for comparisons
  * state_read for reading heap entries
  * state_update for modifying tracked heap state
  * heap_update_per_level for logical O(log s) heap updates
  * state_memory_per_slot only for s heap entries

Correctness requirements:

* Sampling with k < n - s must never scan all n bins.
* Sampling must never include bins currently in the heap.
* When k == n - s, scanning all untracked_bins_ is correct.
* When s == n, untracked_bins_ is empty and all outside sampling/admission logic must be skipped.
* Keep behavior for weighted balls unchanged.
* Do not change PowerKSimulator or StatefulRoundRobinSimulator.
* Keep all code inside namespace balls_bins.
* Prefer clear helper methods over clever code.
